## Summary

Brings operator-deployed Agent telemetry in line with ADL schema v0.19.0 (follow-up from adl-cli #281). Agents deployed via the operator can now configure OTLP/Prometheus trace and metric exporters instead of just an on/off switch.

- Extend the shared `TelemetrySpec`/`MetricsSpec` (`api/v1alpha1`) with `traces.exporter.otlp{endpoint,protocol}` and `metrics.exporter.{otlp{endpoint,protocol},prometheus{host,port}}`.
- Map them onto the Go ADK's `A2A_OTEL_*` env vars in `agent_controller.go`, mirroring adl-cli's `internal/templates/telemetry_env.go`:
  - `A2A_OTEL_TRACES_EXPORTER`, `A2A_OTEL_METRICS_EXPORTER` (`otlp`/`prometheus`/`none`)
  - shared `A2A_OTEL_EXPORTER_OTLP_{ENDPOINT,PROTOCOL}` pair - the Go ADK exposes a single OTLP endpoint, so traces win and metrics fall back (adl-cli does the same)
  - `A2A_OTEL_EXPORTER_PROMETHEUS_{HOST,PORT}`
- Reconcile the enable var: emit `A2A_TELEMETRY_ENABLE` (what the Go ADK actually reads) instead of the never-read `TELEMETRY_ENABLED`.
- Regenerate CRDs, deepcopy and `manifests/{crds,install}.yaml`; update the sample + agent-server example; add controller tests for the mapping.

Note: `TelemetrySpec` is shared with the Gateway CRD, so the gateways CRD gains the same exporter schema. The Gateway controller ignores the new exporter fields (it deploys a different image), so behavior there is unchanged.
