# GPU: externally hosted, GPU-backed inference runtimes

The `GPU` custom resource provisions and manages an externally hosted, GPU-backed
inference runtime reachable over HTTP. The Kubernetes cluster is the **control
plane only** - GPU hardware, drivers, scheduling and inference execution stay
with an external infrastructure provider (RunPod today, pluggable for others).

A `GPU` is **not** a worker node, a device, an `nvidia.com/gpu` request, or a
replacement for the NVIDIA device plugin / Dynamic Resource Allocation.

```text
GPU CR -> Provider API -> External GPU instance -> runtime ready
       -> HTTPS endpoint + connection Secret -> Inference Gateway
```

## Lifecycle

```text
Pending -> Provisioning -> Starting -> Ready
                                    -> Terminating -> Expired
(any step) -> Failed
```

`Ready=True` is reported **only** after the configured `endpoint.readinessPath`
responds successfully - provider infrastructure status alone is not enough.

- The external allocation is tagged with the Kubernetes object UID, so
  reconciliation (including after a controller crash) recovers the existing
  allocation instead of creating a duplicate.
- `maxRuntime` is **required**. When it elapses the allocation is released and
  the resource is parked in `Expired`. An `Expired` resource is not
  reprovisioned automatically; recreate it or change a spec field to lease again.
- A finalizer releases the external allocation on deletion. Provider `Not Found`
  is treated as successful cleanup.

## Apply

```bash
kubectl apply -f gpu.yaml
kubectl -n inference-gateway get gpu qwen-coder -w
```


## Connection Secret

When the runtime is ready the controller creates/updates
`<name>-connection` (e.g. `qwen-coder-connection`):

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: qwen-coder-connection
stringData:
  url: https://abc123-8080.proxy.runpod.net
  apiKey: <generated-per-allocation-token>
```

The `apiKey` is a **per-allocation token generated by the operator**, injected
into the runtime as the `API_KEY` env var. The RunPod management API key is
never exposed here.

## Consuming it from a Gateway

No GPU-specific Gateway wiring exists yet (auto-discovery is out of scope for
this first cut). A Gateway consumes the connection Secret through the standard
provider `env` / `secretKeyRef` mechanism - see the `Gateway` in
[`gpu.yaml`](./gpu.yaml):

```yaml
providers:
  - name: qwen-coder
    enabled: true
    env:
      - name: LLM_BASE_URL
        valueFrom:
          secretKeyRef:
            name: qwen-coder-connection
            key: url
      - name: LLM_API_KEY
        valueFrom:
          secretKeyRef:
            name: qwen-coder-connection
            key: apiKey
```
